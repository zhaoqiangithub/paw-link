## 📝 文档贡献规范

### 文档命名规范

- 使用清晰描述性的文件名
- 中文文档使用中文文件名
- 英文文档使用英文文件名，单词用下划线或短横线分隔
- 示例：`产品需求分析报告.md`, `api_documentation.md`

### 文档格式规范

- 使用 Markdown 格式
- 文档开头包含标题和简要说明
- 使用清晰的层级结构（标题层级）
- 包含目录（对于长文档）
- 代码示例使用代码块并标注语言
- 表格对齐，便于阅读

### 文档更新规范

- 重要更新记录版本历史
- 标注更新日期和更新人
- 保持文档与代码同步更新
- 定期审查和更新过时内容
