## 阶段一：用户体验深度分析

### 1.1 用户角色画像与旅程地图

#### 角色一：救助者（Rescuer）- 核心用户

**基本信息：**
- 年龄：25-45岁（占比70%）
- 性别：女性占75%，男性占25%
- 职业：自由职业者、学生、爱心人士
- 特征：有爱心但缺乏组织化经验

**用户目标：**
- 快速发布需要救助的动物信息
- 获得及时响应和帮助
- 追踪救助进度和结果

**核心旅程：**

```
发现动物 → 初步判断 → 发布求助 → 等待响应 → 志愿者/机构对接 → 
救助执行 → 医疗治疗 → 寻找领养 → 完成领养 → 回访反馈
```

**关键触点：**
1. **发现阶段**（时长：5-30分钟）
   - 场景：路上遇到流浪动物
   - 行动：拍照→判断需不需要帮助
   - 痛点：信息不全（地址描述困难）
   - 需求：快速定位功能

2. **发布阶段**（时长：3-5分钟）
   - 关键行动：填写表单→上传图片→选择紧急程度
   - 最大痛点：表单填写繁琐（现有平台平均需要15分钟）
   - **解决方案**：智能表单+语音输入+地址自动获取

3. **等待阶段**（时长：1-48小时）
   - 用户焦虑：担心动物情况恶化
   - 期望：快速获得回应
   - 痛点：信息石沉大海
   - **解决方案**：实时推送+智能匹配附近志愿者

4. **对接阶段**（时长：30分钟-2小时）
   - 沟通方式：电话/微信/站内信
   - 痛点：身份可信度不确定
   - **解决方案**：身份认证标识+信用评分

**成功指标：**
- 救助响应时间 < 2小时
- 救助成功率 > 70%
- 用户满意度 > 4.5星

#### 角色二：领养者（Adopter）- 核心用户

**基本信息：**
- 年龄：28-40岁（占比65%）
- 性别：男女各占50%
- 职业：有稳定收入的都市白领
- 特征：首次领养或经验领养者

**用户目标：**
- 找到心仪的宠物
- 了解宠物真实情况
- 完成合法领养流程

**核心旅程：**

```
浏览搜索 → 筛选过滤 → 查看详情 → 在线沟通 → 实地看宠 → 
领养申请 → 审核等待 → 签署协议 → 带宠物回家 → 回访
```

**关键触点：**

1. **探索阶段**（时长：5-20分钟）
   - 行为：滑动浏览、筛选、收藏
   - 痛点：信息不真实（占比62%）
   - 期望：快速判断是否喜欢
   - **解决方案**：视频预览+全景照片+健康证明

2. **评估阶段**（时长：30分钟-2小时）
   - 对比多个候选宠物
   - 查看详细信息和救助故事
   - 痛点：无法判断真实性
   - **解决方案**：实名认证+医疗记录+救助历史

3. **沟通阶段**（时长：1-3天）
   - 多轮沟通确认细节
   - 安排实地看宠
   - 痛点：频繁询问基本信息
   - **解决方案**：标准化信息卡+快速问答模板

4. **决策阶段**（时长：1-7天）
   - 家庭成员讨论
   - 准备领养环境
   - 痛点：审核流程不透明
   - **解决方案**：进度追踪+时间预估

**成功指标：**
- 搜索到匹配宠物 < 5分钟
- 有效沟通率 > 60%
- 领养成功率 > 30%（行业平均15%）

#### 角色三：志愿者（Volunteer）- 重要用户

**基本信息：**
- 年龄：22-50岁
- 性别：女性占80%
- 技能：有时间、有爱心，部分有救助经验

**用户目标：**
- 参与有意义的救助活动
- 贡献自己的时间和技能
- 获得成就感和认可

**核心旅程：**

```
注册认证 → 接收任务 → 报名参与 → 执行任务 → 
提交成果 → 获得积分 → 升级权限 → 深度参与
```

**关键触点：**

1. **入门阶段**（时长：10-30分钟）
   - 注册并完成志愿者认证
   - 学习基础救助知识
   - 痛点：认证流程复杂
   - **解决方案**：分步引导+视频教程

2. **接单阶段**（时长：实时）
   - 查看附近任务
   - 根据时间和技能选择
   - 痛点：任务信息不明确
   - **解决方案**：详细任务描述+技能匹配

3. **执行阶段**（时长：2-8小时）
   - 前往现场进行救助
   - 实时更新进展
   - 痛点：现场情况复杂
   - **解决方案**：专家在线指导+应急预案

4. **成长阶段**（长期）
   - 技能提升
   - 获得认证
   - 成为团队负责人
   - 痛点：缺乏晋升路径
   - **解决方案**：积分体系+技能徽章+社区认可

**成功指标：**
- 志愿者活跃率 > 40%
- 任务完成率 > 80%
- 年度留存率 > 50%（行业平均40%）

#### 角色四：救助机构（Organization）- 商业用户

**基本信息：**
- 类型：动物收容所、救助站、宠物医院
- 规模：小型（5-20只）到大型（100+只）
- 需求：提升曝光、增加领养、管理效率

**用户目标：**
- 提高动物领养率
- 扩大品牌影响力
- 降低运营成本
- 获得社会支持

**核心旅程：**

```
注册认证 → 发布动物信息 → 管理库存 → 
推广营销 → 匹配领养者 → 审核申请 → 线下交接 → 回访跟踪
```

**关键触点：**

1. **入驻阶段**（时长：1-3天）
   - 提交资质证明
   - 等待审核
   - 痛点：审核时间长
   - **解决方案**：分批审核+快速通道

2. **运营阶段**（每日）
   - 发布动物信息
   - 回复咨询
   - 更新状态
   - 痛点：工作量大
   - **解决方案**：批量管理+自动更新

3. **推广阶段**（每周）
   - 策划活动
   - 撰写故事
   - 痛点：内容创作困难
   - **解决方案**：模板库+成功案例

**成功指标：**
- 机构入驻数 > 100家
- 平均领养率提升 > 40%
- 机构付费转化率 > 20%

### 1.2 核心交互流程设计

#### 流程一：发布救助信息流程（参考Airbnb发布房源）

```
点击"发布" → 选择类型（救助/领养） → 
自动定位 → 填写基本信息（4步表单） → 
上传照片（1-9张） → 选择紧急程度 → 
确认发布 → 推送通知 → 等待响应
```

**详细步骤：**

**Step 1: 类型选择**
- 页面布局：顶部Tab切换（救助/领养）
- 默认选中：基于用户历史行为智能推荐
- 参考模式：Instagram Story发布选择（照片/视频/其他）

**Step 2: 位置获取**
- 弹出地图（参考滴滴选上车点）
- 智能定位：当前定位+手动调整
- 地址描述：自动填充+手动补充
- 安全提示：可模糊定位（保护隐私）

**Step 3: 基本信息（4步表单）**

*第一步：动物信息*
- 动物类型：猫/狗/其他（大按钮选择）
- 年龄：幼体/成年/老年（滑块选择）
- 性别：公/母
- 品种：输入自动补全
- 健康状况：多选（已免疫/未免疫/患病/未知）

*第二步：救助信息*
- 发现时间：今天/昨天/其他日期
- 发现地点：精确/模糊
- 动物状态：健康/受伤/虚弱/其他
- 紧急程度：低/中/高/紧急
- 需要帮助：医疗/食物/临时寄养/领养/其他

*第三步：详细描述*
- 文本框：自动提示常用描述
- 字数限制：50-500字
- 参考模板："在XXX发现一只X岁左右的X猫，状态XXX，需要XXX帮助"

*第四步：联系方式*
- 必填：手机号（脱敏显示）
- 可选：微信号/其他联系方式
- 是否愿意公开：开启后其他用户可直接查看

**Step 4: 图片上传**
- 参考模式：Instagram多图上传
- 最多9张，第一张为主图
- 支持：拍照/相册选择/拖拽排序
- 智能提示：
  - "建议上传动物全身照"
  - "清晰的照片更容易被领养"
- 自动优化：压缩+添加水印

**Step 5: 发布确认**
- 信息预览卡片
- 重要信息高亮
- 发布按钮：突出显示
- 同步选项：
  - 分享到社区（可选）
  - 推送给附近用户

**技术实现要点：**
- 表单自动保存草稿
- 支持离线编辑
- 图片预上传（加快发布速度）
- 发布后立即推送给附近用户

#### 流程二：搜索匹配流程（参考Tinder滑动匹配）

```
进入搜索 → 地图视图/列表视图切换 → 
应用筛选条件 → 浏览匹配结果 → 
左右滑动预览 → 查看详情 → 
收藏/联系/举报 → 返回继续
```

**详细步骤：**

**Step 1: 搜索入口**
- 位置：首页顶部搜索框
- 默认：智能推荐（基于位置、历史行为）
- 热搜标签：#紧急救助# #免费领养# #已免疫#

**Step 2: 视图切换**
- 顶部Tab：地图视图 / 列表视图
- 列表视图：参考Instagram Explore
- 地图视图：参考Airbnb地图搜索

**Step 3: 筛选器（左侧滑出面板）**

*基础筛选：*
- 距离：1km/3km/5km/10km/全城
- 动物类型：猫/狗/其他
- 年龄：幼体(0-6月)/成年(6月-2岁)/成犬(2-8岁)/老年(8岁+)
- 性别：不限/公/母
- 健康状况：不限/健康/需医疗/未知

*高级筛选：*
- 紧急程度：全部/高/中/低
- 发布时长：今天/3天内/7天内/更早
- 已免疫：需要/不需要/不限
- 品种：输入搜索
- 救助者类型：个人/机构/志愿者

*智能筛选：*
- 推荐算法：基于用户浏览历史
- 个性化标签："你可能喜欢的"
- 相似宠物推荐

**Step 4: 结果展示**

*地图视图：*
- 参考Airbnb地图 + Netflix地图流
- 地图标记：
  - 待领养：绿色图标
  - 需救助：红色图标
  - 紧急救助：闪烁红色
  - 已领养：灰色图标
- 标记聚合：缩放级别 < 14时聚合
- 热力图：救助密度可视化

*列表视图（参考Instagram Explore）：*
- 卡片布局：2列网格
- 卡片信息：
  - 主图（占2/3高度）
  - 标题："2岁金毛弟弟求领养"
  - 位置："距离1.2km"
  - 标签：#已免疫 #已绝育
  - 救助者信息：头像+认证标识
- 加载方式：瀑布流（Infinite Scroll）

**Step 5: 详情页（参考Tinder宠物版）**

*顶部：*
- 图片轮播（1-9张）
- 左右滑动切换
- 底部：保存/分享/举报

*主体信息：*
```
宠物名称：小花
基本信息：2岁母猫 • 已免疫 • 已绝育
位置：朝阳区三里屯（距离1.2km）
发布时间：2小时前
紧急程度：普通
```

*救助故事：*
- 文字描述
- 救助时间线

*健康状况：*
- 疫苗记录
- 医疗证明
- 健康评分

*救助者信息：*
- 头像+姓名
- 认证状态（已实名/已认证）
- 信用评分
- 其他宠物

*快速操作：*
- 收藏（❤️）
- 咨询（💬）
- 电话联系（📞）
- 导航前往（🗺️）

**Step 6: 后续操作**

*收藏：*
- 收藏列表页面
- 价格提醒功能
- 相似宠物推荐

*联系：*
- 弹窗：选择沟通方式
- 站内信/电话/微信
- 快捷短语模板

*分享：*
- 社交媒体分享
- 生成专属海报
- 复制链接

#### 流程三：一对一沟通流程（参考微信聊天）

```
选择沟通对象 → 选择沟通方式 → 
发送消息/图片/位置 → 接收回复 → 
安排线下见面 → 交换联系方式 →

**详细步骤：**

**Step 1: 发起对话**

*入口：*
- 详情页快速联系按钮

- 搜索结果页长按用户

*方式选择弹窗：*
```

[💬] 站内私信（推荐）
[📞] 电话联系

[📍] 发送位置

*顶部导航：*
- 对方头像+名称
- 在线状态
- 认证标识
- 更多按钮（举报/拉黑/举报）

*消息区域：*

**Step 3: 沟通模板（针对不同角色）**

**Step 4: 交换联系方式**

- 双方都有意向

### 1.3 用户痛点与解决方案矩阵

| 痛点 | 影响用户 | 严重程度 | 解决方案 | 参考App |
|------|----------|----------|----------|---------|
| 信息真实性不确定 | 领养者 | ⭐⭐⭐⭐⭐ | 实名认证+健康证明+视频验证 | 支付宝芝麻信用 |
| 救助响应速度慢 | 救助者 | ⭐⭐⭐⭐⭐ | 智能推送+附近志愿者匹配+紧急标记 | 滴滴紧急单 |

### 2.4 数据流设计

#### 数据流一：发布数据流

```
用户输入 → 前端验证 → 草稿保存 → 
提交服务器 → AI审核 → 人工复核 → 
存储数据库 → 推送到其他用户 → 
生成通知 → 更新用户Feed
```

**详细步骤：**

1. **前端数据收集**
   - 表单数据：JSON格式
   - 图片：压缩后上传OSS
   - 位置：GPS坐标

2. **数据验证**
   - 必填项检查
   - 格式验证（手机号、身份证）
   - 图片质量检查

3. **草稿机制**
   - 自动保存到本地（SQLite）
   - 每30秒同步到服务器
   - 跨设备同步

4. **提交处理**
   - POST到 `/api/publish`
   - 返回：发布ID + 状态（审核中）

5. **审核流程**
   - AI初筛：10-30秒
   - 人工审核：5-30分钟
   - 审核通过：状态变为"已发布"

6. **分发推送**
   - 推送给附近用户（地理围栏）
   - 加入推荐算法池
   - 发送到相关群组

7. **数据存储结构**
```json
{
  "id": "pet_123456",
  "type": "adoption/rescue",
  "pet": {
    "name": "小花",
    "species": "cat",
    "breed": "混血",
    "age": 2,
    "gender": "female",
    "vaccinated": true,
    "neutered": true
  },
  "location": {
    "latitude": 39.9042,
    "longitude": 116.4074,
    "address": "北京市朝阳区三里屯"
  },
  "publisher": {
    "user_id": "user_789",
    "verified": true,
    "credit_score": 850
  },
  "status": "published", // draft/reviewing/published/closed
  "created_at": "2025-11-15T10:30:00Z",
  "metrics": {
    "views": 0,
    "likes": 0,
    "shares": 0,
    "messages": 0
  }
}
```

#### 数据流二：搜索数据流

```
用户输入条件 → 前端构建查询 → 
调用后端API → 数据库查询 → 
Elasticsearch检索 → 结果排序 → 
返回前端 → 渲染UI
```

**查询参数：**
```json
{
  "query": {
    "bool": {
      "must": [
        {"term": {"status": "published"}},
        {"geo_distance": {
          "distance": "5km",
          "location": {"lat": 39.9042, "lon": 116.4074}
        }}
      ],
      "filter": [
        {"term": {"pet.species": "cat"}},
        {"range": {"pet.age": {"lte": 3}}},
        {"term": {"pet.vaccinated": true}}
      ]
    }
  },
  "sort": [
    {"_score": {"order": "desc"}},
    {"created_at": {"order": "desc"}}
  ],
  "from": 0,
  "size": 20
}
```

**排序策略：**
1. 相关性分数（个性化推荐）
2. 发布时间（新发布优先）
3. 距离远近
4. 认证用户优先

#### 设计原则

1. **清晰性（Clarity）**
   - 界面元素清晰可见
   - 文案简洁明了
   - 操作结果明确

2. **遵从性（Deference）**
   - 内容优先于装饰
   - 界面服务于功能
   - 避免过度设计

3. **深度感（Depth）**
   - 层级关系明确
   - 动画引导注意
   - 物理隐喻（卡片悬浮）

#### 布局规范

### 4.4 可访问性标准

#### 文本可访问性

**最小字体大小：**
- 正文：17pt（iOS推荐最小值）
- 小标签：12pt
- 确保所有文本可以放大到200%仍可读

**对比度要求：**
- 正文文字：对比度 ≥ 4.5:1
- 大文字（≥18pt）：对比度 ≥ 3:1
- 按钮文字：对比度 ≥ 4.5:1

**动态文字：**

Text("宠物名称")

**图片优化：**

**列表优化：**
- 使用LazyVStack/LazyVGrid
- 可见区域预加载
- 避免在滚动时进行复杂计算
- 分页加载，避免一次性加载全部数据

**网络优化：**
- 请求合并：批量获取宠物详情
- 数据缓存：离线查看已加载数据
- 增量更新：只获取变化的数据
- 压缩请求：使用gzip
