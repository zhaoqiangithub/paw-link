## 🔄 测试流程

### 1. 测试准备
- 阅读 [测试用例文档](./PawLink测试用例文档.md)
- 准备测试环境（设备、数据）
- 了解功能需求

### 2. 测试执行
- 按优先级（P0 → P1 → P2 → P3）执行测试
- 记录测试结果
- 发现问题立即记录

### 3. 缺陷管理
- 提交缺陷到 [测试问题清单](./测试问题清单.md)
- 分配优先级（P0/P1/P2/P3）
- 跟踪修复进度

### 4. 回归测试
- 缺陷修复后重新测试
- 验证修复效果
- 更新测试报告

### 5. 测试总结
- 更新 [测试执行报告](./测试执行报告.md)
- 评估测试覆盖率
- 提出优化建议

## 📝 测试规范

### 测试用例编写规范

**命名规则**:
- 格式: `TC-{模块缩写}-{序号}`
- 示例: `TC-AUTH-001`, `TC-MAP-005`

**必填字段**:
- 测试用例名称
- 优先级（P0/P1/P2/P3）
- 测试类型（功能/性能/安全/兼容性）
- 前置条件
- 测试步骤
- 预期结果

### 缺陷提交规范

**命名规则**:
- 格式: `BUG-{序号}` 或 `WARN-{序号}`
- 示例: `BUG-001`, `WARN-003`

**必填字段**:
- 缺陷描述
- 优先级（P0/P1/P2/P3）
- 复现步骤
- 预期行为
- 实际行为
- 影响范围
- 建议修复方案
